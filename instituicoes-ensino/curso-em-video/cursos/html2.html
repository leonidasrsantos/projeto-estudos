  1. Capítulo 1 - Explicação do módulo e o surgimento da internet:
    • 1958- Criação da Darpa na Guerra Fria;
    • 1969- ARPANET, uma rede criada para enviar dados entre bases militar;
    • 1970- Criação do protocolo NCP para maquinas diferente se comunicar;
    • 1970- Crescimento da rede;
    • 1972- NCP se torna impraticável de usar.
    • 1972- Criação do TCP para eliminar problema do NCP de parar rede para apenas uma comunicação;
    • 1972- Criação do IP para indetificar maquinas;
    • 1983- Abandono dos militares a ARPANET por conta de muitos usuários indevidos (Faculdades, empresas,etc);
    • Década 90- Tim Berners-Lee cria o http, HTML e o www, e Marc Andreessen cria o primeiro navegador, o Mosaic;
  2. Capítulo 2- Funcionamento da internet
    • O computador funciona por sinais de ondas quadradas (sinais elétricos), representados como 0 e 1;
    • o conjunto 0 e 1 se chama binary digit (digítos binarios);
    • O 0 ou o 1 sozinho se chama bit, o minimo para representar algo é o conjunto de 8 bits (01000001 = A) que se chama byte;
    • O computador trabalha com a base 2, então 1KB equivale 1024 bytes, 210= 1024;
    • Multiplos de byte:
      • 1024 bytes = 1KB
      • 1024 KB = 1 MB
      • 1024 MB = 1 GB
      • 1024 GB = 1 TB
      • e por ai vai...
    • MB = megabytes | Mb = megabits
    • Megabyte é para armazenamento, o Megabits é velocidade de transmissão.
    • O modem é responsavel por fazer a onda quadrada virar uma onda senoidal e refazer a onde quadrada ao chegar em outro dispositivo (Modulacão e Demodulação);
    • Toda maquina conectada na internet recebe um número (IP) e ele muda a todo momento;
    • Para encontrar o servidor (instagram,youtube e todos outros sites) temos DNS (ex: Instagram.com)
    • Os dados vem em pacotes diferentes (uma imagem pode ser quebrada em varios pacotes que seguem caminhos diferentes)
    • Os pacotes seguem caminhos diferentes pois muitas vezes um caminho pode estar cogestionados.
    • Domínio:Nome que indentifica o site; Hospedagem: Local que o site será armazenado
  3. Capítulo 3- Diferença de HTML CSS e JS, Front-end Back-end e FullStack
    • HTML: HyperText Markup Language
      • Criado por Tim Berners-Lee
      • Não é uma linguagem de programação
      • Linguagem para marcação de texto
      • Focada em conteúdo
    • CSS: Cascading Style Sheet
      • Folhas de estilos em cascata
      • Servem para mudar o visual
    • JS: JavaScript
      • Linguagem de programação
      • Serve para criar diversas interações
    • Front-End: Responsável por fazer a parte visual e as interações de um site com tecnologias client-side (HTML,CSS,JavaScript...);
    • Back-end: Responsável em fazer a parte de funcionamento do servidor como ligação a bancos de dados com tecnologias server-side (PHP,Java,Python...);
    • FullStack: É um desenvolvedor que domina front-end e back-end;
  4. Capítulo 4- Primeiro codigo
    • <h1>Olá, Mundo!</h1>
  5. Capítulo 5- Tags simples,simbolos e emoji
    • <hr>Usamos para criar uma linha horizontal;
    • <p> Usamos para criar um paragráfo
    • <br>Usamos para quebrar linha
    • Para usar simbolos basta usar & em seguida o codigo do simbolo
    • Para usar emoji basta usar &#x em seguida o codigo do emoji
  6. Capítulo 6- Direitos Autorais e imagens
    • Evite pegar qualquer imagem ou outros arquivos;
    • Existem arquivos gratuitos para livre uso;
    • Existem sites faceis de encotrar imagens gratuitas para uso como o Unsplash e o Pexels;
    • Existe um software gratuito para edição chamado GIMP;
    • Evitar usar imagens pesadas;
    • Usar formato PNG em apenas imagens que necessitam transparência;
    • GIF:Formato que permite transparência e animação;
    • JPEG:Formato mais leve com alta compactação, porém não permite transparência;
    • PNG:Formato que mantém a transparência da imagem;
    • Tamanho maximo de uma imagem para usar em um site é 1500px de largura evitando assim evitando muito peso;
    • Resolução recomendada 72px/in;
    • <img> Serve para adicionar imagens no documento html tanto internas quanto externas;
    • Para melhor desempenho no site, editar imagem para o tamanho certo antes do uso;
  7. Capítulo 7- Hierarquia de Títulos
    • As tags de titulos vai de <h1> até <h6> sendo h1 a mais principal, por ai em diante
    • Não necessariamente pode haver apenas um h1,mas evitar também usar desnecessáriamente.
  8. Capítulo 8- Semântica HTML e Formatação de texto
    • Não usar tags obsolentas como center,font,big. Uma hora elas serão removidas por completo e o editor ajuda identificala.
    • Sempre usar a tag no lugar certo para aquele determinado conteúdo, por exemplo, a tag <address>
      São Paulo, Brasil
      </address> ela não se trata de deixar o texto em itálico e sim a dar um significado a ele;
    • A tag certa para negrito é <strong> pois a tag <b> não tem semântica,dando apenas uma forma e não um sentido, da mesma forma o <em> substitui o <i>;
    • Sempre formar na semântica e não apenas na forma como strong e b onde o visual continua o mesmo, mas uma dá sentido a palavara.
    • Existem outras diversas tag para dar semântica ao conteúdo como:
      • mark: Irá deixar o texto marcado com uma cor;
      • ins: sublinha o texto dando um sentido de preenchimento;
      • sup: Irá deixar o texto sobrescrito x2;
      • sub: Itá deixar o texto subscrito H2;
      • blockquote: irá referenciar uma citação;
  9. Capítulo 9- Listas
    • Lista Ordenadas (ol): Irá demonstrar contagem como 1,2,3 ou A,B,C seguindo a ordem
    • Lista não Ordenadas (ul): A sequência não a uma ordem a seguir.
    • Parâmetros:
      • start: Diz de onde irá iniciar a contagem (ex 2 3 4 ou d f g);
      • type: tipo do marcador antes do conteúdo;
    • Lista de Definição (dl): Dentro da tag (dl) usamos para criar um termo com a tag (dt) e seu significado com a tag (dd)
  10. Capítulo 10- Links
    • Links são importantes pois o algoritimo do Google é baseado em links
    • <a> Usado para links externos e internos.
    • Uso externo: Usado para acessar sites terceiros.
    • Uso interno: Usado para navegar nas paginas do site podendo ser através de menu ou até mesmo algumas palavras marcadas no conteúdo
      • Parâmetros:
      • target: Irá definir onde o link será aberto (na mesma guia ou numa nova ou até mesmo num iframe.);
      • rel: Irá dizer ao mecanismo para onde o site vai levar os usuários ou se existe alguma relação com o endereço por exemplo para um site externo é recomendado usar rel="external";
    • Links para download: Basta colocar os parâmetros: download="nomedoaquivo.formato" type="O media type do arquivo(por exemplo "application/pdf" )"
      • a href="download/docdownload.pdf" download="docdownload.pdf" type="application/pdf">
  11. Capítulo 11- Midías
    • Imagens Responsivas
      • Para criar uma imagem responsiva primeiro precisamos da mesma imagens com dimensões diferente
      • As tags usadas são Picutre em seguida source e o img
      • dentro do picture a tag img deve ficar embaixo dos source e para funcionar deve estar com a menor imagem ou com a maior imagem;
      • Para definir o momento que a imagem se altera usamos no source o parâmetro (media="(max-width= 300px)") e pode ser definido por tamanho maximo "max-width" ou por tamanho minimo "min-width" que a imagem poderá se manter ao site atingi o tamanho determinado;
      • Logo após definir o tamanho, definimos a proxima imagem no source que irá aparecer usando o parâmetro (srcset="localdaimagem/a.png"imagem-media.png");
    • Audios
      • Podemos adicionar audio usando somente a tag (audio src="media/happy.mp3");
      • Para evitar problemas de compatibilidades podemos criar source dentro da tag audios com os 3 principais formatos nos sources, MP3,OGG e WAV;
    • Videos
      • O mesmo padrão do audio mudando apenas a tag para (video)
      • Tomar cuidado com o peso, pois pode afetar o desempenho do site;
      • De preferência usar videos hospedados em sites externos;
      • Para adicionar um video externo basta ir em compartilhar e procurar a opção de incorporar html;
  12. Capítulo 12- CSS
    • Estilo Inline: Se trata do estilo ser adicionado diretamente ao conteúdo daquela linha usando o parâmetro style: <li> style="background:'black';"item</li>
    • Estilo Interno: É colocado a tag style no head logo após o titulo do site, dentro dela irá ser colocado toda configuração css do site
    • Estilo Externo: O jeito mais recomendado por conta de organização, é feito um link de ligação no head com uma pagina totalmente CSS, a onde a pagina vai puxar todas propriedades CSS.
